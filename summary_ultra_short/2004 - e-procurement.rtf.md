# Ultra Short Summary
- This specification defines interoperability requirements for eProcurement message exchanges between public sector buyers and external sellers across the procurement cycle from catalogue to remittance.

**MVP Points**
- Support core procurement documents including Purchase Order, Invoice, and Catalogue.
- Enable electronic document exchange using XML in compliance with e-Government Interoperability Framework.
- Provide common data elements for parties, documents, lines, and items reusable across all documents.
- Allow incremental implementation starting with basic documents like Purchase Order and Invoice.

**Key Constraints**
- Must use standardized identifiers and codes including international standards like ISO country/currency codes and UNSPSC commodity classification.
- Excludes message transport, security, seller discovery, e-auctions, and communication with third parties like banks.
- Documents must include mandatory status attributes (original, copy, revision, cancellation) and UUIDs for unique identification.
