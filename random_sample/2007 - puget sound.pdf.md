# Short Summary

- **Background and objectives**: The University of Puget Sound is evaluating Moodle as a replacement for Blackboard to address instructional gaps and enhance the learning experience through tailored courseware features.
- **In scope**: Multiple file uploads per page, audio recording and portfolio management, web feeds (RSS), course-level search functionality, and gradebook with history tracking.
- **Out of scope**: Not mentioned.
- **Roles and core use cases**:
  - As a student, I want to upload multiple assignment files so that I can submit complex projects efficiently.
  - As a professor, I want to enable web feeds per page so that students receive timely updates on new content.
  - As a system administrator, I want configurable backups so that course data is protected and restorable.
- **Success metrics**: System supports 1000+ concurrent users, achieves 99% uptime, and enables flexible audio clip downloads in MP3 format.
- **Major constraints**: Must use existing Moodle APIs, maintain simple and responsive UI, ensure nightly backups, and require minimal maintenance effort from IT staff.
- **Undecided issues**: Not mentioned.
