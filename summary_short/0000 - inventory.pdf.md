# Short Summary

- **Background and objectives**: IUFA aims to integrate three faculty databases into a unified web-based inventory system, providing secure access for users to manage university assets during working hours.
- **In scope**: Asset transfers, editing/modifying assets, creating/reserving requests, authentication, and generating reports.
- **Out of scope**: Not mentioned.
- **Roles and core use cases**:
  - As a **User**, I want to create requests so that I can borrow assets or reserve spaces.
  - As an **Inventory Administrator**, I want to modify assets so that inventory records are accurate.
  - As an **Administrator**, I want to change user permissions so that tasks can be delegated securely.
- **Success metrics**: System availability during working hours, user learning time under 4 hours, and query response under 1 minute.
- **Major constraints**: Must run on Microsoft/Unix platforms, support major browsers, and restrict database access to IT members only.
- **Undecided issues**: Not mentioned.
