# Short Summary

- **Background and objectives**: The SCM system aims to deliver Ejada company products and services efficiently, with goals including customer service management, procurement, manufacturing flow, and performance measurement.
- **In scope**:  
  - Customer, supplier, and request management  
  - Item and resource location management  
  - Performance measurement and reporting  
  - Web-based user interfaces for three roles  
  - Integration with Ejada’s .NET framework  
- **Out of scope**:  
  - Integration with external CRM or HR systems  
  - Hardware interface requirements  
  - Non-web-based access methods  
  - Non-.NET technology stacks  
  - Mobile or non-browser clients  
- **Roles and core use cases**:  
  - As a coordinator, I want to manage requests, items, and suppliers so that supply chain operations run smoothly.  
  - As a customer, I want to submit and track requests so that I receive products and services on time.  
  - As a supplier, I want to view and respond to requests so that I can fulfill supply orders efficiently.  
- **Success metrics**:  
  - Support for at least 100 concurrent users  
  - 90% of transactions completed in under 1 second  
  - System availability of 100% with understandable error feedback  
- **Major constraints**:  
  - Must use ASP.NET, C#, and MS SQL  
  - Must comply with Ejada’s .NET framework  
  - Must be web-based and accessible via IE or Firefox  
  - Follows Waterfall process and object-oriented design  
  - Must integrate with two other Ejada modules  
- **Undecided issues**:  
  - Communication channels to actors: Not mentioned  
  - Refined request viewing criteria: Not mentioned  
  - Handling of acknowledged request edits: Not mentioned  
  - Supplier/customer notification preferences: Not mentioned  
  - Future external system integration details: Not mentioned
